Embedding: Vector

This page goes through raster image and Vector Image, 分别对于光栅图和矢量图。
Raster image includes png, jpg and gif, which is a representation of image using fixed grid of pixels.
Vector Images contains path and shape definition for the image, which the computer will process and render in situ

Vector: SVG

SVG为基于XML的矢量图格式，有的人会使用adobe illustrator来制造svg files。SVG的好处是可以一体化，矢量图内的每一个元素（包括文字）都是可以单独拆分并且被搜索到的SEO，并且也可以使用css以及js来进行styling。
但是复杂的svg图像需要很大的储存空间以及渲染时间，并且svg不被老版本浏览器支持，创造svg格式也比光栅图要难很多。

嵌入SVG：img

插入svg最快的方式为img，与图片类似。

具体语法与插入普通图片类似，为:
<img src="D:/Programming/Imgs/Starsvg.svg" alt="A hexagonal star with bold blue boundary and green fill" height="87">

Pros

  • 快，简单
  • 可以将svg当做hyperlink使用，用<a>来生成超链接
  • svg可在浏览器内缓存Cache住，以后再次打开便快速加载
Cons

无法使用JS来更改，操作图像。并且需要将css styles内嵌在svg代码内（external reference不起作用）。且不能用CSS pseudoclasses来更改格式

Troubleshooting, Cross-browser support

对于不支持SVG图片的浏览器，可以使用srcset attribute来reference svg图片，支持svg的浏览器将会从srcset来提取，而不支持的可以使用原本的png格式。具体操作详见MDN

Inline SVG

因为svg类似markup langague，不一定非要external ref，可以在html文件内inline defined。确保svg图片由<svg>和</svg> tag开头/结尾。

Pros

  • 将svg设为inline可以节省一次http请求，加快加载速度
  • 可以对svg图片添加属性以及id class, id。因此css可以操作svg图片
  • inline svg是唯一能使用css交互的svg嵌入方法
  • 可以对svg图片使用超链接
Cons
  • 若在多出使用了同一个svg，浏览器会加载多次，比较浪费资源
  • Inline svg会增加html文件的大小
  • 不像普通图片，inline svg在每次html加载都会重复一次，因此并不会缓存下来，以后也不会快多少
  • 若遇到不支持的浏览器，的确可以使用<foreignObject>来生成fallback image，但是即使是支持svg的浏览器仍会下载fallback image，产生资源浪费

另外，还有使用Iframe嵌入svg的方法，但是不推荐使用因为
1.iframe的fallback mechanism不完整，不能针对svg的fallback生成image。
2. 除非svg及网页拥有相同的来源，不然也不能使用JS操作svg
