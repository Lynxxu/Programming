Adding Images on a webpage

加入图片时，我们使用<img>来添加图片，使用width，height,以及figure等attribute来改变/添加属性。

The img tag

<img>类似a(超链接)，里面也有src attribute代表图片的路径。类似之前提到的，路径可以为网上的absolute URL(不建议，因为会使浏览器做无用功），也可以是电脑内图片路径, 比如我引用"D:\Programming\Imgs\2021121623318.png"会有以下结果：

Alternative text alt

img有一个自带的attribute名为alt，这个属性带来的，当某些原因无法正常放出图片时，展示的图片文字描述。这个原因可以是以下几种：

  • Visually impaired user with screenreader
  • wrong spelling & path name
  • browser doesn't support displaying image(text only browser)
  • For search engine optimization
  • 没流量的倒霉人关了图片显示

使用alt attribute时，根据图片目的来决定里面究竟该写什么。 比如以装饰性目的的可以留空(建议直接使用CSS)，对于含有内容的图片，可以留下一段简短的图片描述，或者直接塞在正文内。对于含有链接的图片，在alt内带上链接内容。

Image titles

和链接一样，html可以使用title attribute来展示悬浮窗文字。 但是screenreader并不一定会百分百展示出悬浮窗的内容，所以最好的办法依然是将图片标题带入正文中。

Annotation of images with figures

若想给图片加上解释caption，可以使用最原始的办法，也就是在图片下面带上一段话，如
<div class = "figure">
<img src = "source.jpg">
<p>caption<p>
</div>
 但是很明显，这并不会给caption带来semantics，也就是说accessibility有可能无法解读字幕和图片的关系。

为了解决这个问题，我们有<figute>和<figcaption>。将以上的代码以figure代替，我们有：
<figure>
<img src = "source.jpg">
<figaption>caption<figcaption>
</figure>

figure不一定非得是图片，它可以是一块independent content，占据大块内容，提供关键信息并且有效的解释你的想法，如下:

  • code snippet
  • audio
  • video
  • table

CSS background image

前面提到，在html内添加图片的主要目的是解释内容。而对于装饰性的图片，我们使用css来添加。语法如下：

        p{
            background-image: url("yoursourceofimage.jpg");
        }
