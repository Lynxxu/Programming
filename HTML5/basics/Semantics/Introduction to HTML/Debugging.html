Debugging html

Debug,即纠错，此文档提供一些纠错的方式

Debugging

不同与java，python等编程语言，浏览器解析(parse)HTML时更加宽容(permissive)，意思是即使我们写出了错误，浏览器依旧可以解析并展出结果。

当我们写出bug时，一般有以下两种：

  • syntax errors:即语法错误，比如spelling或punctuation error。 一般来说写错了语法会导致程序无法运行，但是因为html的permissive nature，即使有语法错误依旧可以运行（但可能不是你想的那样）
  • logic errors：逻辑错误，即语法正确，程序可以运行，但是不是你想的那样运行，有可能会是错误的指引，变量问题等等。
