Describing the UI

React 为一个JS库，专门用于渲染UI。React可以将许多小的units，比如buttons，images，texts加在一起，变成reusable nestable components。我将在这个页面学习react component的创建，定义和显示等操作。

First component

React applications are built from small isoalted pieces of UI called components. A react component is a JS function which can be changed, it can be small as a buttonm and large as an webpage.

Using HTMl, markup structures like a ordered list can be created, combined with css and JS. React would combine all the css, js and HTML into custom components that are resulable UI elements. Some open source communitites already have thousnads of components to be used like MUI

Define a component

Traditionally, web devs would write html, markup and add interaction via JS. It is now cumbersome and difficult if it were to be repeated thousands of times. React makes them reusable, but at the end of day, a React component is a JS funciton that you can sprinkle wiht markup. A block of code like below would generate 

        export default function Profile() {
          return (
            <img
              src="My waifu"
              alt="Noelle"
            />
          )
        }

Step 1: Export the component

export defautl is a standard JS syntax that allows the funciton to be exported to other fiels to be used. See later sections

Step 2: define function

Using function Profile() {}, this defines a JS function with name Profile. Note that a React component must start with CAPITALIZED letter, unlike regular functions

Step 3: Add markup

as written, this component would return an img tag with scr and alt attribute. It is like a html bnut it's acutally JS. This syntax is called JSX, which enables markup inside JS.

the return statement can also be writte with one line.Like:

return <img src='source.jpg' alt ='some alt text'/>

However, if the markup spans over multiple lines, it must be included in a pair of parentheses: 

      return (
        <div>
          <img src = 'sth' alt='sth'>
        </div>
      )

At here, a component was finished, with fist line, enabling export of the component to other files. Defined using function Profile(){} like a noraml JS function but capitalized. Which is then implemented with a markup that can return a img with specified attributes

Using a component

Now we defined a Profile component. It can be added to a html as if it's a part of it. However, there are several things to do before adding it:

  • Remeber to import react script tags like react.development.js or via a importmap
  • Import Babel as complier so that browser can read JSX and React syntax
  • Note in the react js file type="text/babel"

Then, we can nest the React functions inside one other. The Profile function is finished and can be nested like below: 

      export default function Gallery(){
        <section>
          <h1>Amazing Fuhua>
          <Profile/>
          <Profile/>
          <Profile/>
        <section>
      }

and we would have the following effect:

In browser, this process is quite straightforward. <section> is lowercase so it's a HTML tag, but <Profile> is capitalized so we are calling a component called Profile. Therefore, in browser, they are equivalent to the codes below 

    <section>
      <h1>Amazing Fuhua>
      <img src="../../../../Imgs/83181495_p0.jpg" alt="Fuhua/>

Another thing to be mentioned is that, to nest components, a component should NOT be defined in another component. It should be separatedly declared

Importing and exporting components

As mentioned in the previous part, we made a component named Profile that can be exported and saved in component.js. For reasons like resuability, we can write the module in separate files for management. To do this we can use the import and export functions

For example, we can create a Gallery js file, which includes both the Profile and Gallery method. Remeber to include export defualt for the intend export file

Then, in another separate js file, we can write the following code to import the method: 

    import Gallery from './Gallery.js'

    export default function App(){
      return (
        <Gallery />
      );
    }

The logic in here is that Gallery.js defines the Profile component to be used internally in Gallery (not exported). Then exports the Gallery component as default export. In App.js, the Gallery is imported as a default import from Gallery.js, while the root App component is exported default

Export and import in the same file from multiple files

What if I wish to import both the Gallery and Profile method in one file. This can be done using a mix of default export and named export. It is important to note that there is only one default export per file. Therefore, The rest of the method needs to be named export. In the export statement, they simply remove the default keyword. In the import statement, they add a pair of curly braces in the name of the imported component.

So, in Gallery.js, the code will be updated as:

    export funciton Profile(){
      return (
        <img
          src = "some directories"
          alt = "alt tetxt"
        />
      );
    }

    export default function Gallery (){
      return(
        <section>
          <h1>Amazing fuhuas </h1>
          <Profile />
          <Profile />
          <Profile />
        </section>
      );
    }

in the App.js file, where both components are being imported:

      import Gallery from './Gallery.js'
      import { Profile } from './Gallery.js'

      export default function App(){
        return(
          <Profile />
        );
      }

The relation between files are simply yet intertwined. The Gallery.js exports both component, one with default one with name. In App.js, the Gallery component is imported as default, Profile is imported as name component with curly braces. Later, it exports App.
