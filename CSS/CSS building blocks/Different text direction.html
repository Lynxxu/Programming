Handling different text directions

之前学到的内容基本上都是基于屏幕的物理尺寸来的，比如使用top，left等指向来代表box的某条边。而css以及浏览器默认是基于left-to-right语言的支持。

但是在近几年， css对于其他书写方式的支持变得越来越高了，而且不仅是从右到左（阿拉伯语），还有从上到下（日语），这些书写方式被称为writing mode

Writing modes

在css内，writing mode代表了文字的方向，也是一个属性。比如我可以使用writing-mode: vertical-rl让这句话从上到下

writing-property可以接受以下几个值：

  • horizontal-tb：block从上到下，而句子由水平方向走（英语）
  • vertical-rl：block从右到左，（所以是rl），句子vertical走

    类似这样(这是另一个block

  • vertical-lr:block从左到右，句子vertical

    也就是和rl相反的

当我们改变writing mode的时候，其实我们顺便改变了block以及inline的配置。当使用默认的mode时，inline方向即是句子的方向（从左到右）， 而block的方向则是从上到下。但是当我们改成vertical-rl时，inline则变成了从上到下的方向，而block则是从right到left，vertical-lr同理。

Logical properties and values

之前讲过了如何调整border，margin等等box的性质，但是他们都是基于horizontal的书写模式，也就是基于电脑的物理屏幕尺寸。

但是当使用vertical modes时，原本的box特性比如width,height 依旧有效，但是对于vertical来说却是相反的。因此css在最近发明了新的办法：logical，or flow relative properties,当使用这些新的properties时，box的尺寸是相对于writing mode来说的，而不是对于电脑屏幕的左右。具体对应的如下：

  • inline-size: inline的尺寸，一般来说是从左到右，对于vertical来说是从上到下。（相应于width）
  • block-size:block的尺寸，对于一般的mode来说就是从上到下的尺寸。对于vertical来说则是从左到右

Logical margin, border and padding properties

之前讲过可以使用inline-size以及block-size来调整不同writing mode 的width和height。但是肯定还有其他的box内容相对应的，比如padding-left，margin-top等等。具体来说如下：

  • margin-block-start:对应margin-top，即从block开始的方向的margin
  • padding-inline-start:对应padding-left ，即从文字的方向（inline的方向）开始的padding
  • border-block-end：即对应border-bottom，指代border的结束位置，（因为border为block）

之前学的默认设定的内容被称为physical，而这里描述的新内容称为logical properties。记得logical为一个相对新的内容，有很多浏览器也许不支持，因此若不使用多种书写方式时，建议依然使用physical的。

之前看的都是logical properties，但是有一些properties也可以接受logical values。对应如下：

  • block-start对应top
  • block-end对应end
  • inline-start对应left
  • inline-end对应right

例子：

  • border-top对应border-block-start
  • margin-left对应margin=inline-start
  • padding-bottom对于padding-block-end
