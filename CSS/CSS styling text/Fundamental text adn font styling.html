fundamental text and font styling

Styling text: what's involved

在之前学过了那么多的css和html基础后，我们了解到text应该是在一个元素的content box内的，从左到右（或者从右到左，对于部分语言来说）直到一行结束。text content实际上就是一系列的inlne元素挨个的放在一起并且不生成line break（除非分行）。

style text使用的css元素一般能分为两类，如下：

  • Font styles:影响text的字体，比如什么字体，尺寸，黑体，斜体等等属性。
  • Text style styles: 影响spacing以及其他的text layout相关，比如字母间距，和哪里对齐等等。

Web safe fonts

当谈到字体时，要知道并不是所有的字体都可以在所有机器上使用，所以一般来说会使用几种基本上所有操作系统都支持的字体，他们被称为web safe fonts

在绝大多时候，网页开发者希望对字体有更高的权限，但是找到全平台通用的字体并不简单，所以为了通用性，一般会使用web safe fonts，可以使用CSS fontstack来查找web safe fonts

Font stacks

因为我们不可能100%保证某个字体是可以在全平台无障碍使用的，所以我们可以创造一个font stack来给浏览器提供不止一个字体。使用的便是font-family 

            div{
                font-family "Trebuchet MS", Verdana, Arial, sans-serif
            }

以上的代码便是给div元素提供了四个字体，浏览器会自动从头遍历这四个字体，并且使用第一个可以使用的。因此建议最好在最后一个字体提供generic name，这也即使前面所有的字体都不可用，还是会有一个generic有效的。假设全部都失效，那么浏览器会自动使用Times New Roman

另外提一句，类似Times New Roman, Trebuchet MS这种含空格的字体，需要使用quotes将其包围起来

Font size

在前面的css values and units内学过，font-size可以接受绝大多数的单位，但是一般来说使用最多的单位如下:

  • px:像素，为绝对单位，不论在什么情况和什么设备，基本上都是相同的尺寸
  • em: 1 em就是母元素的font size，更详细一点来说就是parent element内的M字母的宽度。使用em在nested elements内会变得比较tricky，但是为什么不呢，基本上所有东西都可以使用em来指定尺寸，甚至可以将整个网页都使用em来决定尺寸以此保证maintainability
  • rem: 大体来说和em的逻辑相同，但是1rem是与root element的font size有关，也就是<html>，而不是母元素。这样就变得更简单了，因为不需要在nested元素内一直算来算去的。但是需要注意的是很多老浏览器是不支持rem的，比如IE8和以下

一个元素的font-size是自然继承母元素的，但是整个文档的一切均始于<html>，也就是默认的16px尺寸。绝大多数未被指定尺寸的元素均为16px，部分比如<h1>之类的，会以几em来作为默认尺寸 

            <article>
                <p&gr Nested elements</p>
            </article>

以上便是一个简单的nested paragraph，当article和p均使用em来设置尺寸时，p会受html以及article的影响，而这种影响在更深的嵌套元素内会变得错综复杂，所以尽可能的避免设置font-size，而在必须使用的时候可以用rem来保持简洁

Font style, font weight, text transform and text decoration

css提供了四种常见的属性来更改text的weight/emphasis

  • font-style: 用于打开/关闭italic text，可以接受三个值：
    • normal:也就是正常，或者关闭italic
    • italic: 将字体设置为italic verision（如果有的话），若字体没有意大利体，则切换为oblique来模拟italic
    • oblique: 将正常字体倾斜以模拟意大利体
  • font-weight: 设置黑体的宽度，若字体内含许多不同的variants，则可以接受许多值。但是实际上来说基本上只会用得上normal以及bold
    • normal, bold：分别对应normal和bold的font-weight
    • lighter, bolder:将现元素的黑体程度设置更低/或更高
    • 100-900:也就是数字值，控制的比较准确
  • text-transform: 可以调整字体的transformation
    • none：即没有transformation
    • uppercase:将所有的text转换成大写字母
    • lowercase:将所有text转化为小写字母
    • capitalize: 将每一个单词的首字母大写
    • full-width: 将每一个字符的大小调整一致，类似monospace字体。这种办法可以使得拉丁语系和亚洲语系对齐
  • text-decoration: 设置/取消文字的装饰，接受以下几种值:
    • none: 取消一切decoration
    • underline: 将字体下划线
    • overline: 将文字上划线
    • line-through: 从字体中间划线

需要注意的是，text-decoration可以接受多个值，并且它其实是一种shorthand，代表的是text-decoration-line, text-decoration-style 以及text-decoration-color。所以说可以一次性加上很多种decoration，

Text layout

学完了基本的字体style，下面来讲一下基本的layout属性。

Text alignment

text-alignment属性可以控制文字与其的box的对齐程度，可以接受left， right, center,justify这几个值，分别对应了左右居中对齐和占据全box（类似word里面的justify)

Line Height

line-height属性可调整每一行文字的高度，其可接受绝大多数的css length和size unit，并且也可接受unitless value，比如1，2，3，4，此时他们作为乘子被用于和font-size相乘

类似在word里面，double space也就是line-height:2; 一般来说建议调整line height至1.5和2之间。

Letter and word spacing

letter-spacing以及word-spacing分别对应了调整字母和单词之间的间距，虽然不会继经常用到这些，但是有的时候会有用，比如需要给某个特别dense的字体提高可读性，

Text layout styles

  • text-indent: 调整每一段开头的缩进长度
  • text-overflow: 调整溢出内容的显示方法（比如放被卡住的内容，用...代替等等），与overflow属性不同
  • whitespace: 定义空格以及分段的特性，比如要不要每一句话单独空行，空行开头要不要缩进之类的
  • word-break: 对于可能会跨行的单词，调整跨行的属性
  • direction: 也就是文字的方向
  • hyphen：调整跨行单词的特性，是否使用hyphen来链接
  • line-break: 调整分行的标准（对于亚洲语言）
  • text-align-last: 定义一个block的最后一行话的alignment
  • text-orientation: 定义文字在一行内的方向
  • Overflow-wrap: 定义浏览器能不能在文字之间分行以避免溢出
  • writing-modes: 文字的方向，是从左到右还是从上到下，定义的是行的方向

Font shorthand

上面写了许多关于font的属性，但是字体的这些属性还可以通过一种shorthand，也就是font来决定

font接受的值按顺序是font-style，font-variant, font-weight, font-stretch, font-size, line-height, font-family。需要注意的是，font属性最少需要font-size以及font-family。且font-size和line-height之间需要使用/隔开
