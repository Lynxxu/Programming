Web fonts

在第一篇文章内，我学过了基本的css文字装饰。现在来学的更细节一点。

Font families recap

往html内加上字体的办法是使用font-family属性如下：

        html{
            font-family: Impact, Helvetica, "Times New Roman", sans-serif;
        }

上面的办法非常有用，但是类似之前提到的，开发者并不能随便找个字体都可以跨平台无障碍的使用。一般来说，在一个font stack内，你想要的字体优先级最高，但是建议加上一款web safe alterntive，在加上默认的浏览器字体。虽然这种办法很保险，但是会增加工作量，因为你需要测试每一个字体和网页的适配性

Web fonts

除了使用font-family以外，还有一个非常好用的办法，他甚至支持IE6这种老浏览器。具体来说，他可以让用户打开网页时下载你指定的字体文件（如果可以的话），意思就是只要支持该功能的浏览器都可以显示你指定的字体。

具体来说，它由@font-face开头，写在css的开头处，类似一个这样的语句： 

        @font-face{
            font-family: "someFonts";
            src: url("the file for the font")

        }

然后，就像平常一样，使用font stack来指定刚刚@rules内的字体

        html{
            font-family: "someFonts", "Bitstream Vera Serif", serif;
        }

虽然语法看起来很简单，但是有很多东西需要注意的：

  • 字体不总是免费的，有的时候需要付钱或者credit原作者
  • 绝大多数浏览器都支持WOFF/WOFF2 (Web Open Font Format version 1 & 2)，即使是比较老的浏览器也能支持WOFF
  • WOFF2 支持全部的TrueType和OpenType specification。包括variable font， chromatic font和font collection
  • 列举字体文件的顺序很重要，如果给浏览器提供了多个字体文件，它会选择第一个能用的字体，所以建议将最想要的字体列在第一位（WOFF2），然后老浏览器便可以支持后面的老字体
  • 若希望支持已过时的浏览器，应该提供EOT(Embedded Open type), TTF(TrueType Font)以及svg格式的字体以供下载。下面会讲一下如何使用工具来生成这些文件
